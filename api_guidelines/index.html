    Правила использования SWAPI

    Внимательно прочитайте эти правила во избежание потери доступа к StupidWallet API (далее 'API'; 'REST API'; 'SWAPI').

    1. Конфиденциальность

    • 1.1. StupidWallet - крипто-платформа, у которой в приоритете конфиденциальность и защита данных пользователей. По этой причине ВСЕ сервисы, использующие API, обязаны неукоснительно соблюдать настоящие Правила и обеспечивать полную сохранность и безопасность данных пользователей.
    • 1.2. Запрещается несанкционированное вмешательство в функционал любой из двух подтвержденных сетей StupidWallet (Testnet и Mainnet).

    2. Прозрачность

    • 2.1. Для использования API требуется ключ, регистрируемый отдельно для каждого приложения.
    • 2.2. Наш API предоставляется бесплатно всем разработчикам и энтузиастам, поэтому пользователи Вашего сервиса должны быть осведомлены об использовании API.
    • 2.3. Во избежание путаницы, Ваш сервис не должен содержать в своем названии "StupidWallet", или слово "official".
    • 2.4. Ваш сервис не должен использовать логотип StupidWallet в качестве своего.
    • 2.5. Разработчик сервиса обязан собирать статистику и историю транзакций при любых обстоятельствах и предоставлять администрации доступ к этим данных.

    3. Монетизация

    • 3.1. Ваш сервис может взимать комиссию от транзакций не более 3%.
    • 3.2. Если вы собираетесь как либо монетизировать сервис (включая комиссию), у Вас должно быть личное согласие проверяющего/администрации.

    4. Нарушение правил

    • 4.1. Если Ваш сервис, использующий SWAPI, нарушает любой из вышеуказанных пунктов, на Ваш сервис будут наложены различные ограничения транзакций и вывода. Функционал, нарушающий Правила должен быть удален из сервиса в течение недели после уведомления.
    • 4.2. Если Вы не исправите проблемы, доступ к API будет ограничен практически полностью, а сервис будет остановлен.

    Заключение

    • Мы (StupidWallet Foundation) оставляем за собой право расширять, изменять содержание Правил по мере возникновения необходимости.
